# 03 刷新机制与排查

## 1. 刷新链路（两种机制）

桌面小组件展示更新依赖 **两条互补路径**：

```text
路径 A（数据变）：主 App 写快照 → WidgetKitBridge.reload* → Provider.getTimeline 重跑
路径 B（时间到）：GardenFlowerLoader 预生成未来 Entry → 到点 WidgetKit 切换，无需再 push
```

| 步骤 | 位置 | 说明 |
|------|------|------|
| 1 | 游戏 / JS | `widget/default` request/update |
| 2 | `WidgetMgr` | merge v3 栏位 → `widget/widget_snapshot.json` |
| 3 | `WidgetReloadThrottler` | 防抖后 `reloadGardenWidgets()` |
| 4 | `GardenFlowerProvider` | `GardenFlowerLoader.timeline(for:)` |
| 5 | `GardenFlowerCardView` | 读 `GardenFlowerDisplayModel` 渲染 |

珍珠/公会写各自快照后走**同一条** `reloadGardenWidgets()`（6 kind 全刷）。

## 2. 种花 Timeline 策略（`GardenFlowerLoader`）

### 2.1 无快照 / 无栏位数据

- 单条 `Entry`，`date = now`，展示空态（`GardenFlowerWidgetCopy` 占位）。

### 2.2 种植栏 `status == 1` 且 `timeBar.thumb > 0`

- 计算 `endDate = sectionUpdatedAtMs + thumb * 1000`。
- Timeline 在 `now` 与 `endDate` 各一条 Entry：进行中 → 完成（`status` 映射为完成态 UI）。
- 若 `endDate` 已过，直接完成态单条。

### 2.3 花艺出售栏

- 逻辑同种植：剩余秒来自 `timeBar.thumb`，锚点 `sectionUpdatedAtMs`。

### 2.4 水滴栏

- `bar.thumb` < `bar.track` 且 `timeBar.track`（间隔秒）> 0：为每一滴恢复生成边界 Entry（圆环进度递增）。
- 已满滴：静态 Entry，无未来点。

### 2.5 Small vs Medium

- **Small**：`compactCountdownOnly`，仅映射 `flowerPlant`。
- **Medium**：三栏并行映射，`WidgetSectionColumn` 三等分。

### 2.6 倒计时 UI

- iOS 16+：`Text(timerInterval:countsDown:)` 绑定 `WidgetCountdownSnapshot`。
- 颜色：`bottomTxt.style.color` 或 `WidgetTokens.Text.countdown` / 种植主题色。

## 3. reload 行为

```swift
// WidgetKitBridge.reloadGardenWidgets()
reloadAllTimelines()
reloadTimelines(ofKind: "HYGardenFlowerSmall")
reloadTimelines(ofKind: "HYGardenFlowerMedium")
reloadTimelines(ofKind: "HYPearlCollectSmall")
// … 珍珠、公会共 6 kind
```

注意：

- 系统可能合并短时间多次 reload；**不要**在 1s 内连发数十次 request。
- reload 只保证「尽快重新拉快照」；**到点变 UI** 仍靠 Timeline 未来条目。
- 修改 Swift 代码后需重新编译 Extension；仅改 JSON 也需 reload 或等 Timeline 到期。

## 4. 常见问题

### 4.1 小组件不更新

| 检查项 | 处理 |
|--------|------|
| App Group 未 configure | 主 App 启动时 `WidgetMgr.configure` |
| 写入非 v3 | 必须 `{ type, data }`，看返回 `status` |
| Extension 未装最新 | 删小组件重装 / 升版本号 |
| reload 过密被吞 | 拉大业务推送间隔或 `setReloadMinInterval` |
| 快照路径错 | 确认容器内 `widget/widget_snapshot.json` 存在且 `schemaVersion == 3` |

### 4.2 倒计时卡住或快跳

| 现象 | 原因 | 处理 |
|------|------|------|
| 一直 00:00 | `sectionUpdatedAtMs` 未更新或 thumb=0 | 每次 push 带当前毫秒锚点 |
| 完成态不出现 | 未生成 end Entry | 确认 `status=1` 且 thumb>0 |
| 与游戏差几秒 | 主 App 节流 / 系统刷新延迟 | 以 Timeline 边界为准验收 |

### 4.3 图片不显示

- Asset 名拼写与 `Assets.xcassets` 一致。
- `http` 图：主 App 是否已下载并写入 App Group 映射。
- Extension 日志搜 `WidgetImageLoader` / 栏位 `img.normal`。

### 4.4 中英文不对

- **无快照**：看系统首选语言（`GardenFlowerWidgetLocale.isPrimaryChinese`），非地区。
- **有快照**：以服务端 `topTxt`/`bottomTxt` 为准。

### 4.5 颜色不对

- 确认 JSON `style.color` 为 `#RRGGBB`。
- `{$time}` 行走倒计时色逻辑，不是静态 `topTxt` 色。

## 5. 调试手段

1. **query**：`widget/default` query 或 `WidgetMgr.querySnapshot()` 看磁盘 JSON。
2. **Mock**：`PluginAdapter_widget` 内 `sections` 数组（多栏一次写入）。
3. **删快照**：删除 App Group 下 `widget/widget_snapshot.json` 验证空态。
4. **日志**：Extension 进程过滤 `GardenFlower`、`WidgetMgr`、`HYWG`（若工程已打点）。
5. **时间**：修改系统时间仅建议测 Timeline 边界；生产以真实推送为准。

## 6. 与 WidgetKit 系统限制（提醒）

- 刷新频率由系统调度，非实时推送。
- 后台主 App 被挂起时，仅靠已写入快照 + 预生成 Timeline。
- 用户关闭「小组件刷新」或低电量模式可能延迟 reload。
