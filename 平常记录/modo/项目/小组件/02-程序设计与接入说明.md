## 2. 快照文件（App Group）

目录：`{AppGroupContainer}/widget/`

| 文件 | 管理类 | schema |
|------|--------|--------|
| `widget_snapshot.json` | `WidgetMgr` | **v3**（`flowerPlant` 等四键，按栏 merge） |
| `pearl_snapshot.json` | `PearlCollectMgr` | v1 |
| `guild_competition_snapshot.json` | `GuildCompetitionMgr` | v1 |

写入流程：`encode` → `{name}.tmp` → `move` 覆盖，避免 Extension 读到半截 JSON。

App Group 默认规则（未 `configure` 时）：`group.{主AppBundleId}`，Extension 通过 `guessHostBundleId` 去掉 `.HYWidgetExtension` 等后缀对齐主包。

## 3. 种花 v3 协议

### 3.1 写入形态（唯一合法）

```json
{
  "type": "flower_plant",
  "data": {
    "status": 1,
    "topTxt": { "text": "种植中", "style": { "color": "#333333" } },
    "bottomTxt": { "text": "{$time}", "style": { "color": "#E85D4A" } },
    "img": { "normal": "widget_flower_planting", "completed": "widget_flower_done" },
    "timeBar": { "thumb": 120, "track": 0 },
    "sectionUpdatedAtMs": 1716000000000
  }
}
```

- `type`：`flower_plant` | `flower_sale` | `flower_water_drop` | `flower_competition`（见 `WidgetGardenSectionType`）。
- `data`：统一 `WidgetGardenSectionPayload` 结构。
- **调试多栏**：`{ "sections": [ { "type", "data" }, ... ] }`（`PluginAdapter_widget` 顺序 apply）。

非 v3（缺 `type`/`data`、或旧扁平字段直传）→ `WidgetMgr` 返回 `v3PacketRequiredResponse`。

### 3.2 栏位必填（解析层）

| type | 必填 | 说明 |
|------|------|------|
| `flower_plant` | `status` | 种植 |
| `flower_sale` | `status` | 花艺出售 |
| `flower_water_drop` | `bar` | `thumb` 当前滴数，`track` 上限 |
| `flower_competition` | `topTxt` | 竞赛（按需接入 UI） |

### 3.3 倒计时与水滴

- **种植/花艺进行中**：`timeBar.thumb` = 剩余秒数；锚点 `sectionUpdatedAtMs`（毫秒）。
- **水滴恢复**：`timeBar.track` = 每滴间隔秒数；`bar.thumb` / `bar.track` = 当前/上限滴数。
- 展示层 `{$time}` 模板由 `WidgetCountdownSnapshot` 与 `Text(timerInterval:)`（iOS 16+）驱动，**不是**静态 `bottomTxt.text` 字面量。

### 3.4 文案与颜色

- `WidgetTextItem`：`text` + 可选 `style.color`（`#RRGGBB`）。
- 倒计时颜色：优先 `bottomTxt.style.color`；种植中可叠加主题色逻辑（见 `GardenFlowerMapping`）。
- 无 `style.color`：回退 `WidgetTokens.Text`（`WidgetDesignTokens.swift`）。

### 3.5 图片

- `img.normal` / `img.completed`：栏内 Asset 名或 http key。
- `http` 开头：主 App 下载后写入 App Group，`WidgetImageLoader` 用 `UserDefaults(suiteName:)` 查本地路径。

## 4. JS / Native 插件 API（种花）

| opt | 说明 |
|-----|------|
| `request` | 写入/merge 单栏（或 sections） |
| `update` | 同 request（语义由业务区分） |
| `end` | 清除指定栏或整份种花快照（见实现 `clearSection` / `clear`） |
| `query` | 读当前 `WidgetSnapshot` |

珍珠：`PearlCollectMgr` — `pearlCount`、`slots[]`（`countdownSeconds` / `countdownTime`）、`buttonTitle` 等。

公会：`GuildCompetitionMgr` — `score`、`taskTitle`、`progressA/B`、`eventEndAtMs` 等。

启动时主 App 应调用：

```objc
[WidgetMgr.shared configureWithAppGroupId:@"group.xxx"];
[PearlCollectMgr.shared configureWithAppGroupId:@"group.xxx"];
[GuildCompetitionMgr.shared configureWithAppGroupId:@"group.xxx"];
```

## 5. Widget Extension 接入

1. Target：`HYWidgetExtension`（或宿主等价扩展 target）。
2. `@main`：`WidgetUIDesktopBundle` → `allDesktopWidgets()` 注册 6 个 Widget。
3. Pod：`WidgetUI` + `WidgetCommon`；`pre_install` 动态库策略与主工程一致（见宿主 Podfile）。
4. App Group：主 App 与 Extension **相同** `group.xxx`；entitlements `com.apple.security.application-groups` 一致。
5. `Info.plist`：`NSSupportsLiveActivities` 与小组件无关，但同一扩展工程若含 Live Activity 需单独按 [live-activity 文档](../integrations/live-activity/main-app-integration.md) 验收。

种花代码目录：`WidgetUI/FlowerGarden/`（Models / Mapping / Loader / Widgets / View / Locale）。

## 6. 刷新触发

任意 `*Mgr` 成功写盘后：

```swift
WidgetKitBridge.reloadGardenWidgets()
// reloadAllTimelines + 6 个 ofKind
```

节流：`WidgetReloadThrottler.minInterval` 默认 1s。

Extension 侧 `GardenFlowerLoader` 另根据 `timeBar` 生成未来 `Timeline` 条目（见 [03-刷新机制与排查.md](./03-刷新机制与排查.md)）。
