# 04 测试用例

## 1. 环境准备

- [ ] 真机 iOS 16+（倒计时 `timerInterval`）。
- [ ] 主 App 与 HYWidgetExtension 同一 App Group、已 `configure`。
- [ ] 桌面已添加：种花 Small + Medium；按需珍珠/公会。
- [ ] 可访问 `widget/default` Mock 或测试服推送。

## 2. 种花 v3 — 写入与 merge

| ID | 步骤 | 预期 |
|----|------|------|
| G-01 | 仅 `flower_plant` request | JSON 含 `flowerPlant`，其它栏为空 |
| G-02 | 再 `flower_sale` update | `flowerSale` 更新，`flowerPlant` 保留 |
| G-03 | 传无 `type` 的旧包 | `status: fail`，磁盘不变 |
| G-04 | `sections` 三栏一次 Mock | 三键均有数据，Medium 三栏同屏 |
| G-05 | query | 与磁盘 `widget_snapshot.json` 一致 |

## 3. 种花 — 展示

| ID | 步骤 | 预期 |
|----|------|------|
| G-10 | 无快照冷启动 | Small/Medium 占位中/英文随系统语言 |
| G-11 | 种植 status=1，thumb=60 | 倒计时走动，约 60s 变完成 UI |
| G-12 | 种植 status=2 | 完成插图 + 完成文案 |
| G-13 | 花艺栏出售中 | 中栏倒计时与结束切换 |
| G-14 | 水滴 bar 2/5，track 间隔 30s | 圆环进度，约 30s 增 1 滴 |
| G-15 | Small | 仅左栏种植信息，无中右栏 |
| G-16 | `topTxt.style.color` | 标题色与配置一致 |
| G-17 | `bottomTxt.style.color` + `{$time}` | 倒计时颜色一致 |

## 4. 刷新与稳定性

| ID | 步骤 | 预期 |
|----|------|------|
| R-01 | request 后立刻看桌面 | 数秒内更新（允许系统延迟） |
| R-02 | 1s 内连续 10 次 request | 不崩溃，最终态正确（可能合并） |
| R-03 | 杀主 App，等种植倒计时结束 | 小组件仍自动变完成（Timeline） |
| R-04 | 删小组件再添加 | 仍读同一份快照 |
| R-05 | 升级 Extension 版本后 | 布局正常，无旧 Large 残留 |

## 5. 珍珠 / 公会（冒烟）

| ID | 步骤 | 预期 |
|----|------|------|
| P-01 | pearl request 带 slots 倒计时 | 对应 kind 刷新，槽位走动 |
| P-02 | pearl end | slots 清空 |
| U-01 | guild request 进度与分数 | Small/Medium 展示一致 |
| U-02 | guild end | 进度归零 |

## 6. 异常与边界

| ID | 步骤 | 预期 |
|----|------|------|
| E-01 | 错误 App Group | 写入 fail，小组件空态 |
| E-02 | `sectionUpdatedAtMs` 为未来 | 倒计时不异常爆炸（按实现校正或立即完成） |
| E-03 | 水滴 thumb>=track | 无恢复 Timeline，满滴 UI |
| E-04 | 系统语言切 English | 无快照占位变英文 |

## 7. 回归清单（发版前）

- [ ] 种花无 Large target / 无旧 kind 引用。
- [ ] 文档 [02](./02-程序设计与接入说明.md) 中 v3 字段与实现一致。
- [ ] 与灵动岛联调项：**不**假设 activity 自动更新小组件。
