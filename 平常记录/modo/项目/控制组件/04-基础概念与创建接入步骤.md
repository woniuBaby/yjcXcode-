# 控制组件：基础概念与创建接入步骤

## 1. ControlWidget 是什么

`ControlWidget` 是 iOS 18 的系统控制能力，按钮可出现在控制中心、锁屏等位置。与桌面小组件同属 WidgetKit，但职责是 **一次点击动作**，不是持续展示业务 UI。

| 能力 | 用途 |
|------|------|
| 桌面小组件 | 展示状态（种花倒计时等） |
| 灵动岛 | 长驻活动过程 |
| **Control Widget** | 系统入口 → 打开 App + 传递 `intentName` |

本仓库中：**种花展示** 在 `WidgetUI`；**控制中心** 在 `HYControlWidget` + `ControlWidgetCore`，二者分开 Extension。

---

## 2. AppIntent 是什么

用户点击控制按钮后，系统执行绑定的 `AppIntent.perform()`。

当前推荐链路：

```text
用户点击
  → 宿主 AppIntent（双 Target）
  → ControlWidgetCoreHostedJump.perform
       ├─ 可选：写 App Group ControlWidget[intentName]
       └─ OpenURLIntent(业务 URL)
  → 系统拉起主 App
  → 游戏 control/query、getInfo
```

**难点**：Intent 类型若只定义在 Pod 内，真机常见 **点击无反应、不跳 App**。必须在宿主工程用 **同一 Swift 文件** 声明 `AppIntent`，并勾选主 App + 控制中心扩展。详见 SDK `控制中心-宿主双Target-Intent-必读-防踩坑.md`。

`AppIntent.title` 必须是 **编译期字面量**（如 `"jump"`），不能写 `rawValue`。

---

## 3. TTL 是什么

`TTL`（Time To Live）限制控制点击标记的有效时间，避免用户很久以后手动打开 App 仍被当成「刚从控制中心进来」。

当前 Native 实现：**30 秒**（`PluginAdapter_control` 的 `kMaxAge`），读取时过期项会从 `ControlWidget` 目录删除。

## 4. 创建控制中心 Extension（步骤）

### 4.1 新建 Target

1. Xcode → New Target → **Widget Extension**（或仅控制用途的 Extension）。
2. Deployment Target：**18.0**。
3. 命名如 `HYControlWidgetExtension`；**不要**与桌面小组件混用同一 Bundle 除非团队明确接受 18.0 下限。

### 4.2 Pod

主 App 与扩展 Podfile：

```ruby
pod 'ControlWidgetCore', :path => '…/HYWidget/ControlWidgetCore'
```

### 4.3 宿主 Swift（双 Target）

新建 `HYControlWidgetOpenApp.swift`，勾选 **主 App + 扩展**：

- `HYControlWidgetIntentName`（业务 ID）
- `HYControlWidgetConfig`（URL、展示名、kind）
- 每个按钮：`struct XxxIntent: AppIntent` + `struct XxxControl: ControlWidget`
- `HYControlWidgetMainAppWire.wire()`

复制花园 `build/ios/proj/HYControlWidget/HYControlWidgetOpenApp.swift` 改包名与 URL。

### 4.4 扩展 Bundle（仅扩展 Target）

`HYControlWidgetBundle.swift`：

```swift
@main
struct HYControlWidgetBundle: WidgetBundle {
  var body: some Widget {
    HYControlWidgetJumpControl()
    HYControlWidgetJumpWriteControl()
  }
}
```

### 4.5 主 App 配置

- `CFBundleURLTypes`：与 `HYControlWidgetConfig.openURL` scheme 一致。
- App Group：主 App + 扩展 plist `NTControlCenterAppGroupSuite` = `group.<主包BundleId>`。
- 链接 **AppIntents.framework**；设置 **SWIFT_VERSION**。
- 启动调用 `HYControlWidgetMainAppWire.wire()`（`ControlWidgetCoreBootstrap` 也会尝试反射调用）。

### 4.6 游戏接入 NativeBility

注册 `control` 插件后，在主场景 ready：

```javascript
// 伪代码
const q = await native.call('control', 'default', { opt: 'query', data: {} });
if (q.fromControlCenter) {
  const info = await native.call('control', 'default', { opt: 'getInfo', data: {} });
  switch (info.intentName) {
    case 'jump': /* … */ break;
    case 'jumpWrite': /* … */ break;
  }
}
```

## 5. 新增一个控制按钮（清单）

与 [05-控制中心顺序链路](./05-控制中心顺序链路.md) §4 一致：

1. `HYControlWidgetIntentName` 增加 case  
2. `HYControlWidgetSymbols` + `HYGardenControlDefinitions`  
3. 复制 Intent + Control，`title` 字面量 = case 名  
4. `HYControlWidgetBundle` 增加一行 Control  
5. `wire()` 增加 `_ = 新Intent.self`  
6. 游戏 `getInfo.intentName` 分支  
7. 真机：**删除旧控制中心控件再添加**

## 6. 自检（发版前）

- [ ] `ControlWidgetButton(action:)` 为 **宿主** Intent，非仅 Pod `ControlWidgetCoreButtonIntent`  
- [ ] 宿主 Intent 文件 **双 Target**  
- [ ] URL、App Group 主/扩一致  
- [ ] 点击后 `[control] query 命中` 或 `getInfo` 有 `intentName`  
- [ ] 改代码后已删控件重装
