# 控制中心：最小接入与按项目配置

适用于 **花园（HuaYuann）** 及复用 **`ControlWidgetCore`** 的宿主。扩展须在 **扩展 Target** 内提供 **`@main` `WidgetBundle`**（花园：`HYControlWidgetBundle.swift`）。

> 先读：[05-控制中心顺序链路](./05-控制中心顺序链路.md)、SDK 目录 `HYWidget/ControlWidgetCore/控制中心-宿主双Target-Intent-必读-防踩坑.md`。

---

## 0. 版本与 Deployment

| 能力 | 系统下限 | 花园扩展 Deployment | 说明 |
|------|----------|---------------------|------|
| **控制中心 Control Widget** | iOS 18+ | **18.0**（`HYControlWidgetExtension`） | `ControlWidgetCore` podspec 为 12.0，API 用 `@available(iOS 18.0, *)` |
| **桌面小组件** | iOS 14+ | 以 `HYWidgetExtension` 为准 | 见 WidgetUI，与控制中心 **分开 Extension** |
| **灵动岛** | iOS 16.1+ | 以 Live Activity Extension 为准 | 与控制中心无代码依赖 |

**结论**：Extension Target 的 **Deployment** 表达真实能力下限；Podfile 扩展 target **不必**单独写 `platform`（可继承主工程 12.0）。

## 1. 最小接入清单（新 App）

### 1.1 工程

1. 新建 **Widget Extension**（仅控制中心），Extension **Deployment = 18.0**。
2. Extension **Embed** 到主 App；链接 WidgetKit、SwiftUI、AppIntents（Pod 通常会带入）。
3. **Pod**：主 App + 扩展均 `pod 'ControlWidgetCore', :path => '…/HYWidget/ControlWidgetCore'`。
4. **扩展 `@main`**：自建 `WidgetBundle`，`body` 里注册 **宿主** `ControlWidget`（见花园 `HYControlWidgetBundle.swift`）。
5. **宿主 Swift**（双 Target）：配置 enum + **每按钮一个宿主 `AppIntent` + `Control`** + `MainAppWire`（见花园 `HYControlWidgetOpenApp.swift`）。

**不推荐（真机高风险）**：扩展 Bundle 仅写 `ControlWidgetCoreOpenAppControl<YourDefining>()`，按钮 action 为 Pod 内 `ControlWidgetCoreButtonIntent`。

**可选**：单按钮 demo 可用 `ControlWidgetCoreOpenAppToggle()` + 扩展 plist 的 `ControlWidgetCore*` 键。

### 1.2 主 App

1. 链入 **ControlWidgetCore**（与扩展同版本）。
2. 链接 **AppIntents.framework**；配置 **SWIFT_VERSION**（若有 Swift）。
3. **`CFBundleURLTypes`**：与宿主 `openURL`（如 `huayuan6746321359://openFromControlCenter`）一致。
4. **App Group**：主 App 与扩展 plist 配置 **`NTControlCenterAppGroupSuite`**（或依赖 SDK 从 Bundle ID 推导 `group.<主包ID>`）。
5. **wire**：链 Pod 后 **`ControlWidgetCoreBootstrap` `+load`** 会调 `ControlWidgetCoreWireHostAppIntentsIfNeeded()`（反射 `HYControlWidgetMainAppWire` 等）；`AppDelegate` 再调一次无害。

### 1.3 扩展 Info.plist

- **必填倾向**：`NSExtension` → `NSExtensionPointIdentifier` = `com.apple.widgetkit-extension`。
- **App Group**：`NTControlCenterAppGroupSuite` = `group.<主包 Bundle ID>`（与主 App 一致）。
- **业务配置**：花园在 **Swift**（`HYControlWidgetConfig`）维护 URL、文案、kind；非必须写 `ControlWidgetCore*` plist 键。

## 2. 花园工程配置项（代码侧）

| 配置 | 位置 | 说明 |
|------|------|------|
| `HYControlWidgetIntentName` | `HYControlWidgetOpenApp.swift` | 业务 ID，与 App Group / `getInfo` 的 `intentName` 一致 |
| `HYControlWidgetConfig.openURL` | 同上 | 打开主 App URL，与主 App `CFBundleURLTypes` 一致 |
| `hostBundleIdentifier` | 同上 | **运行时**从 `Bundle.main` 推导主包 ID（扩展会剥 `.HYControlWidgetExtension`） |
| `controlWidgetKindSegment` | 同上 | kind 中间段，默认 `HYControlWidget` |
| `displayNamePrefix` | 同上 | 控制中心列表名前缀，如 `花园·jump` |
| `HYControlWidgetSymbols` | 同上 | 各按钮 SF Symbol |
| `HYGardenControlDefinitions` | 同上 | 每按钮一条 `openAppDefinition(intent:symbolName:)` |
| `AppIntent.title` | 各宿主 Intent | **编译期字面量**，须与 `IntentName` case 同名（不能写 `rawValue`） |

### 2.1 仅在使用 `ControlWidgetCoreOpenAppToggle` 时的 plist 键

| Key | 说明 |
|-----|------|
| `ControlWidgetCoreOpenURL` | 打开主 App 的 URL |
| `ControlWidgetCoreKind` | Control `kind`，全局唯一 |
| `ControlWidgetCoreButtonTitle` | 按钮文案 |
| `ControlWidgetCoreDisplayName` | 控制中心列表名 |
| `ControlWidgetCoreSymbolName` | SF Symbol |
| `ControlWidgetCoreIntentName` | 可选，与 buttonTitle 二选一作 intentName |

`ControlWidgetCoreIntentTitle` / `ControlWidgetCoreIntentDescription`：**当前 SDK 未读取**（`AppIntent` 元数据须编译期字面量）。

## 3. SDK 对外能力（摘要）

| API | 用途 |
|-----|------|
| `ControlWidgetCoreControlCenterLaunchMarker` | 写/读 App Group 约定（`ControlWidget` 目录） |
| `ControlWidgetCoreHostedJump.perform` | 写盘（可选）+ `OpenURLIntent` |
| `ControlWidgetCoreOpenAppDefinition` + `OpenAppDefining` | 按钮参数模型（宿主填配置） |
| `ControlWidgetCoreOpenAppControl` | 可选控件模板（**真机建议改用宿主 Control**） |
| `ControlWidgetCoreWireHostAppIntentsIfNeeded()` | 主 App wire（见 `ControlWidgetCoreHostWire.h`） |

---

## 4. 与桌面小组件

- **WidgetUI**：仅桌面小组件，**无** Control Widget / AppIntents。
- **推荐**：控制中心单独 Extension；桌面小组件另一 Extension。
- **勿**在同一 `WidgetBundle` 混大量桌面 Widget 与多个 Control Widget。

---

## 5. 游戏层（NativeBility）

- 插件：`control` / `PluginAdapter_control`
- API：`query`（`fromControlCenter`）、`getInfo`（`intentName`/`ts`）、`delegate`（清盘）
- App Group 根键 **`ControlWidget`**，子键 = 业务 `intentName`；TTL **30s**（见 [02](./02-程序设计与接入说明.md)）
- 调试：`query` / `delegate` 参数 **`{}`**；测 `getInfo` 过滤可传 `{ intentName: "jumpWrite" }`
- 日志：Console 过滤 **`[control]`**

更细排障见 [07-花园工程跳转主App与排障](./07-花园工程跳转主App与排障.md)。
