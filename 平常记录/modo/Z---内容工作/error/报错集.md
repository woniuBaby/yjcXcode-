2.ArgumentError - [Xcodeproj] Unable to find compatibility version string for object version `70`.

你在执行 `pod install` 时遇到的 `object version 70` 错误，通常是因为项目的 `project.pbxproj` 文件版本过高，而当前环境的 **CocoaPods** 或 **Xcodeproj** 组件版本较低，无法识别该版本号所致

```
🔧 修改 objectVersion 的详细步骤

1. 找到 project.pbxproj 文件：
在你的项目文件中，找到后缀为 .xcodeproj 的文件。
在 macOS 上，右键点击该文件，选择 “显示包内容” 。
在打开的文件夹里，找到并双击名为 project.pbxproj 的文件。建议用文本编辑器（如 VSCode、Sublime Text 甚至系统的文本编辑）打开它。
2. 修改 objectVersion 的值：
在 project.pbxproj 文件中，搜索 objectVersion 。
你会看到类似 objectVersion = 70; 的代码行。
将这个值从 70 修改为一个较旧且被支持的版本号。根据你的 CocoaPods 版本，主要有两种选择：
对于已停止维护的旧版 CocoaPods：建议修改为 63 。
对于较新的环境（如 Xcode 16.2）：可以尝试修改为 77 。
修改后记得保存文件。
3. 重新执行命令：
回到终端，再次运行 pod install 命令。
```
