# 双系统证书同步 — 操作步骤

> **分工：低版本 macOS 管证书，高版本只导入，禁止在高版本新建开发证。**  
> 关联：[iOS证书.md](./iOS证书.md)

---

## 先记住 3 件事

1. **低 → 高，只传 `.p12`（开发证 + 私钥）**，这是必须做的。
2. **Profile（描述文件）一般不用手动导出**——走自动签名时，高版本导入 p12 后让 Xcode 重新下载即可。
3. **发布证（Distribution）不要 Revoke**，App Store 线上包不受影响；开发证换了，真机上的旧 Debug 包要删了重装。

---

## 一、低系统：导出 P12

### 1. 确认本机有一张可用的开发证

1. 打开 **钥匙串访问** → 左侧 **登录** → 分类 **我的证书**
2. 找到 `Apple Development: 你的名字 (TEAM_ID)`
3. 左边有小三角 ▶，展开后能看到 **专用密钥**（私钥）

没有私钥 → 去 Xcode → Settings → Accounts → 你的 Apple ID → Manage Certificates → 点 `+` → **Apple Development** 创建一张（**只在低系统做，高系统禁止点 +**）。

### 2. 后台只留 1 张开发证（多余的可 Revoke）

1. 打开 <https://developer.apple.com/account/resources/certificates/list>
2. 筛选 **Development**
3. 把**旧的多余** `Apple Development` 点 **Revoke**（只 Revoke 开发证，**不要动 Apple Distribution**）

### 3. 导出 P12（核心步骤）

1. 钥匙串 → **我的证书** → 选中 `Apple Development: …`（选证书那一行，不是只选私钥）
2. 菜单 **文件 → 导出"Apple Development: …"**
3. 格式选 **个人信息交换 (.p12)**
4. 保存，例如 `AppleDevelopment-20260709.p12`
5. 设一个导出密码（高系统导入时要输入）
6. 拷贝到外置盘 / 共享目录，带到高系统

### 4. Profile 要不要一起导？

| 项目签名方式 | 要不要手动导 Profile |
|--------------|----------------------|
| **自动签名**（推荐） | **不用**。高系统导入 p12 后，Xcode 会重新拉描述文件 |
| **手动签名** | **要**。去后台重新 Generate 描述文件，下载 `.mobileprovision` 带到高系统（旧 Profile 绑的是旧开发证，已失效） |

> 自动签名 = Target → Signing & Capabilities → 勾选 **Automatically manage signing**。  
> 大多数工程走这条，**低系统只导 p12 就够了**。

## 二、高系统：导入 P12 + Xcode 怎么设

### 1. 清理旧开发证

1. 钥匙串 → **登录** → **我的证书**
2. 删除已失效 / 带红叉的 `Apple Development` 及其私钥
3. **保留** `Apple Distribution` 和 Apple 中间证书，别删

### 2. 导入 P12

1. 双击 `AppleDevelopment-xxx.p12`
2. 钥匙串选 **登录**
3. 输入导出密码
4. 确认「我的证书」里新证旁边有小三角，含私钥

### 3. Xcode 账号

1. **Xcode → Settings → Accounts**
2. 登录**同一个** Apple ID，选**同一个 Team**
3. 点 **Manage Certificates** → 应看到和低系统**同一张** Apple Development
4. **禁止点 `+` 新建开发证**

### 4. 签名：自动 vs 手动（二选一，看工程现状）

#### A. 自动签名（默认走这条）

**不用手动选证书，不用手动选 Profile。**

对每个 Target（主 App、Extension、Widget）：

1. **Signing & Capabilities**
2. 勾选 **Automatically manage signing**
3. **Team** 选你的团队
4. 其余证书 / Profile 由 Xcode 自动匹配（用的是刚导入 p12 里的那张开发证）

然后做一次全局刷新：

1. **Xcode → Settings → Accounts** → 你的账号 → **Download Manual Profiles**
2. 低系统也做同样一步（两台保持一致）

Signing 栏无红色报错即 OK。

#### B. 手动签名（仅当工程关闭了自动签名时）

**要手动选 Profile；证书不用单独下拉选，由 Profile + 钥匙串里的 p12 决定。**

对每个 Target：

1. **取消** Automatically manage signing
2. **Provisioning Profile** 下拉 → 选对应 Target 的描述文件  
   - 若列表里没有 / 报错 → 说明 Profile 还绑旧证，去后台 Edit → 换绑新开发证 → Generate → 下载 `.mobileprovision` → 双击安装
3. **Signing Certificate** 一般显示 `Apple Development: … (自动)` 或灰色不可改——只要 p12 已导入且 Profile 绑对新证即可

手动签名时，**低系统除了 p12，还要把新 Profile 文件带到高系统**（或两台各自从后台下载同一份）。

### 5. 验证

终端执行：

```bash
security find-identity -v -p codesigning
```

应看到 1 条有效的 `Apple Development: …`，无 `Revoked`。

---

## 三、真机：删旧包重装

开发证换了，**已装的 Debug 包验签失败会闪退**，换证救不了，必须：

1. iPhone **长按删除**旧 App
2. Xcode 选真机 → **Run**（⌘R）重新安装
3. 若提示不受信任：**设置 → 通用 → VPN 与设备管理** → 信任开发者

---

## 四、发布证（Archive / 上架时才需要）

- 日常真机调试：**只要开发证 p12**，不用动发布证。
- 高系统要打 Archive / 上传 App Store：高系统钥匙串里也要有 **Apple Distribution** 私钥。  
  - 若只在低系统有 → 在低系统同样方式导出 `AppleDistribution-xxx.p12`，高系统导入。  
  - **不要**在高系统新建或 Revoke 发布证。

---

## 流程一览

```
低系统
  └─ 钥匙串导出 Apple Development.p12
  └─ （手动签名才需要）后台下载新 .mobileprovision

高系统
  └─ 导入 .p12
  └─ Xcode 同一 Team，禁止新建开发证
  └─ 自动签名 → 勾选 Automatically manage signing → Download Manual Profiles
     手动签名 → 每个 Target 选手动 Profile
  └─ 真机删旧 App → 重新 Run
```

---

## 踩坑速查

| 现象 | 处理 |
|------|------|
| 高版本又弹「Choose a certificate to revoke」 | 高版本误点了 Manage Certificates 的 +，禁止在高版本建证 |
| 签名报 Profile 不匹配 | 自动签名 → Download Manual Profiles；手动签名 → 后台重新 Generate Profile |
| 能装但一点就闪退 | 删旧 App 重装 |
| 只有 .cer 没有私钥 | 无法签名，必须从有私钥的机器导 p12 |
