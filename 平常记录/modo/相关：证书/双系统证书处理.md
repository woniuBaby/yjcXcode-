# 双系统（低版本 + 高版本 macOS）开发证书统一处理

> **策略：低版本 macOS 作为主证书管理机**  
> 适用场景：同一台 Mac 双系统启动，共用同一 Apple Developer 账号，需统一开发证书、避免互相吊销。  
> 关联文档：[iOS证书.md](./iOS证书.md)

---

## 0. 背景与问题

### 发生了什么

- 在**低版本 macOS** 上开启 Xcode 自动签名时，**重新生成了一张开发证书**。
- 导致**高版本 macOS** 上原先使用的开发证书失效。
- 以前通过 Xcode 直装到真机的 **Debug / Development 包** 打开即闪退。
- **App Store 已上架版本不受影响**（Apple 会用自家证书重签后再分发）。

### 核心结论（务必记住）

| 问题 | 答案 |
|------|------|
| 吊销「开发证书」会让「发布证书」失效吗？ | **不会**。二者是独立证书类型。 |
| App Store 线上版本会挂吗？ | **不会**。 |
| 真机调试包为什么闪退？ | 旧包装的是旧开发证书链，验签失败。需删 App 重装。 |
| 双系统能共用一张开发证书吗？ | **可以**。低版本导出 `.p12`，高版本导入即可。 |
| 发布证书需要双系统同步吗？ | 若高版本也要打 TestFlight / 上架包，建议同样导出 Distribution `.p12` 备份并同步。 |

### 证书类型对照

| 类型 | Apple 后台名称 | 用途 | 本次是否动它 |
|------|----------------|------|--------------|
| 开发证书 | Apple Development | Xcode 真机调试、开发包 | ✅ 统一为 1 张 |
| 发布证书 | Apple Distribution | 上传 App Store、TestFlight 正式包 | ❌ 不吊销、不重建（除非确认已丢私钥） |
| 企业分发 | Apple Distribution (In-House) | 企业内测直装 | ⚠️ 吊销会导致已装 App 立刻失效，慎动 |

## 1. 总体策略

```
低版本 macOS（主）                    高版本 macOS（从）
─────────────────                    ─────────────────
✅ 唯一允许创建/续期开发证书           ❌ 禁止自动新建开发证书
✅ 导出 .p12 备份                     ✅ 只导入 .p12，不点 Manage Certificates 的 +
✅ 维护 Development Profiles          ✅ 导入同一 .p12 + 刷新 Profiles
✅ 真机调试、iOS 12 兼容测试            ✅ 日常开发、Archive、上传
```

**原则：全账号只保留 1 张有效的 Apple Development 证书。**

Apple 对开发证书上限通常为 **2 张**。双系统若各建各的，极易顶满上限并触发「选择一张吊销」对话框，再次引发连锁问题。

## 2. 执行前准备

### 2.1 确认当前所在系统

低版本为主操作机，以下步骤默认在**低版本 macOS** 执行，标注「高版本」的步骤切换到高版本后再做。

### 2.2 需要访问的页面

- 证书列表：<https://developer.apple.com/account/resources/certificates/list>
- 描述文件：<https://developer.apple.com/account/resources/profiles/list>
- 设备列表：<https://developer.apple.com/account/resources/devices/list>

### 2.3 备份目录建议

在外置盘或共享位置创建：

```
/Volumes/你的备份盘/modo-certs/
├── AppleDevelopment-YYYYMMDD.p12      # 开发证书（含私钥）
├── AppleDistribution-YYYYMMDD.p12     # 发布证书备份（可选，建议做）
├── profiles/                          # 手动签名时的 .mobileprovision
└── README.txt                         # 记录 p12 密码存放位置（不要写明文密码进仓库）
```

### 2.4 本地检查命令

在低版本终端执行：

```bash
# 查看本机可用于签名的身份
security find-identity -v -p codesigning
```

正常输出示例：

```
1) XXXXX "Apple Development: Your Name (TEAM_ID)"
   1 valid identities found
```

若出现 `(CSSMERR_TP_CERT_REVOKED)` 或 `0 valid identities found`，说明本机证书已废，需走「阶段 1 方案 B」重建。

## 3. 阶段 0：盘点（约 5 分钟）

### 3.1 Apple 开发者后台

登录后逐项勾选：

**开发证书（Development）**

- [ ] 当前有几张 `Apple Development`？
- [ ] 哪张是低版本刚生成的（看创建日期）？
- [ ] 旧的是否已显示 `Revoked`？
- [ ] **确认 `Apple Distribution` 仍为 Valid，未 Revoked**

**描述文件（Profiles）**

- [ ] Development 类型 Profile 绑的是哪张开发证书？
- [ ] 项目涉及的 Bundle ID 是否都有对应 Profile（含 Extension / Widget）？

**本项目相关 Bundle ID（示例，以 Xcode 实际为准）**

- 主 App：`HuaYuann` 对应 Bundle ID
- Extension：`HYWidget`、`HYControlWidget` 等

### 3.2 判断走哪条路径

| 情况 | 走路径 |
|------|--------|
| 低版本 `security find-identity` 有 1 张有效开发证书 | **方案 A**：保留低版本新证 |
| 低版本也无有效证书，或不确定 | **方案 B**：Revoke 全部 Development 后仅在低版本重建 |
| Distribution 也被误 Revoke | 先停，单独处理发布证书（见文末「异常处理」） |

## 4. 阶段 1：低版本确立「唯一主证书」

### 方案 A（推荐）：保留低版本已生成的新开发证书

#### 步骤 1：清理多余的旧开发证书（仅 Development）

1. 打开 <https://developer.apple.com/account/resources/certificates/list>
2. 筛选 **Development**
3. 对**已确认不再使用**的旧 `Apple Development` 点 **Revoke**
4. **不要**对 `Apple Distribution` 点 Revoke

> ⚠️ Revoke 开发证书后，绑定旧证的 Development Profile 会失效，阶段 3 会重建。

#### 步骤 2：确认低版本 Xcode 只有一张开发证书

1. 打开 Xcode
2. **Xcode → Settings…**（或 Preferences）→ **Accounts**
3. 选中你的 Apple ID → 右侧 **Manage Certificates…**
4. 确认 **只有 1 张** `Apple Development`
5. **不要**再点左下角 `+` 新建

#### 步骤 3：确认钥匙串中有私钥

1. 打开 **钥匙串访问**（Keychain Access）
2. 左侧选 **登录**，分类选 **我的证书**
3. 找到 `Apple Development: …`
4. 左侧有小三角 ▶ 且展开后能看到 **专用密钥**（私钥）

若没有私钥，说明证书不完整，改走方案 B。

#### 步骤 4：导出 .p12（最关键）

1. 在「我的证书」中选中 `Apple Development: …`（选证书，不是只选私钥）
2. 菜单 **文件 → 导出"…"**
3. 格式选 **个人信息交换 (.p12)**
4. 保存为：`AppleDevelopment-YYYYMMDD.p12`
5. 设置导出密码（务必记住，同步给高版本导入时用）
6. 拷贝到备份目录（外置盘）

验证导出是否有效（可选）：

```bash
# 查看 p12 内证书信息（会提示输入 p12 密码）
openssl pkcs12 -in AppleDevelopment-YYYYMMDD.p12 -nokeys -clcerts | openssl x509 -noout -subject -dates
```

---

### 方案 B：两边都乱，从零只在低版本建 1 张

仅在方案 A 无法用时执行。

#### 步骤 1：Revoke 所有 Development 证书

1. 开发者后台 → Certificates → Development
2. **Revoke 全部** `Apple Development`
3. 再次确认 **未动 Distribution**

#### 步骤 2：仅在低版本创建新证

1. 低版本 Xcode → Settings → Accounts → Manage Certificates
2. 点 `+` → **Apple Development** → 等待创建完成
3. 钥匙串确认有证书 + 私钥

#### 步骤 3：导出 .p12

同方案 A 步骤 4。

## 5. 阶段 2：同步到高版本 macOS

> 切换到**高版本 macOS** 后执行。

### 步骤 1：清理高版本旧开发证书（建议）

1. 钥匙串访问 → 登录 → 我的证书
2. 删除已 **Revoked** 或带红叉的 `Apple Development` 及其私钥
3. **保留** `Apple Distribution` 和 Apple WWDR / G3 中间证书

### 步骤 2：导入主证书 .p12

1. 双击 `AppleDevelopment-YYYYMMDD.p12`
2. 钥匙串选 **登录**
3. 输入导出时设置的密码
4. 确认「我的证书」里新证书旁有小三角，含私钥

### 步骤 3：Xcode 账号对齐

1. Xcode → Settings → Accounts
2. 登录**同一 Apple ID**、选**同一 Team**
3. Manage Certificates → 应看到与低版本**同一张**开发证书
4. **禁止**再点 `+` 创建新的 Apple Development

### 步骤 4：验证签名身份

```bash
security find-identity -v -p codesigning
```

期望：至少 1 条有效的 `Apple Development: …`，无 Revoked 字样。

### 步骤 5：（可选）同步发布证书

若高版本需要 Archive / 上传 App Store，且发布证书私钥只在某台机器上：

1. 在**持有 Distribution 私钥**的那台机器导出 `AppleDistribution-YYYYMMDD.p12`
2. 在另一台导入
3. 同样：**不要重复创建，不要无故 Revoke**

## 6. 阶段 3：重建 / 刷新描述文件（两台都要）

旧 Development Profile 绑定的是旧开发证书，必须更新。

### 方式一：自动签名（推荐）

**低版本、高版本各做一次：**

1. 用 Xcode 打开 `HuaYuann.xcworkspace`
2. 每个 Target（主 App、Extension、Widget）→ **Signing & Capabilities**
3. 勾选 **Automatically manage signing**
4. 选择同一 **Team**
5. Xcode → Settings → Accounts → 你的账号 → **Download Manual Profiles**

### 方式二：手动签名

1. 开发者后台 → Profiles → 各 Development Profile → **Edit**
2. 证书改为**新的 Apple Development**
3. **Generate** → 下载 `.mobileprovision`
4. 双击安装，或放入：

   ```
   ~/Library/MobileDevice/Provisioning Profiles/
   ```

### 本项目检查清单

- [ ] 主 App Target 签名正常（无红色报错）
- [ ] `HYWidget` 签名正常
- [ ] `HYControlWidget` 签名正常
- [ ] `NativeBility`（若本地 target）签名正常
- [ ] Build Settings 里 `DEVELOPMENT_TEAM` 与两边一致

---

## 7. 阶段 4：修复真机闪退（必做）

已安装的旧开发包**无法**通过换证书修复，必须重装。

### 每台测试 iPhone / iPad

1. **长按删除**闪退的旧 App（含相关 Extension）
2. 用 USB 连接 Mac，Xcode 选该设备
3. 用**新证书环境**重新 **Run**（⌘R）安装
4. 首次安装若提示不受信任：**设置 → 通用 → VPN 与设备管理** → 信任开发者

### 若仍闪退

| 现象 | 处理 |
|------|------|
| 安装失败，签名错误 | 检查 Profile 是否已刷新、Team 是否选对 |
| 能装但一点就退 | 确认已删旧 App；查看 Xcode → Devices 崩溃日志 |
| 提示设备未注册 | 开发者后台添加 UDID，刷新 Profile |

## 9. 长期规则（防止再踩坑）

### 9.1 角色分工

| 系统 | 允许 | 禁止 |
|------|------|------|
| **低版本（主）** | 创建/续期开发证书、导出 .p12 | 无故 Revoke Distribution |
| **高版本（从）** | 导入 .p12、Download Profiles | Manage Certificates 里点 `+` 新建开发证 |

### 9.2 证书变更时的同步流程

```
低版本创建/续期 → 导出 .p12 → 拷贝到备份盘 → 高版本导入
                → 两台 Download Profiles → 真机重装 App
```

### 9.3 定期备份（建议每月或每次续期后）

- `AppleDevelopment-当前.p12`
- `AppleDistribution-当前.p12`（如有）
- 关键 `.mobileprovision`（手动签名时）

### 9.4 团队扩展（可选）

若有多人 / CI，建议引入 **fastlane match**，证书存私有 Git，所有人 `match development` 拉同一份。见 [iOS证书.md](./iOS证书.md) 相关章节。

## 11. 异常处理

### Q1：误 Revoke 了 Distribution 怎么办？

- **App Store 已上架版本**：不受影响，仍可从商店下载。
- **影响**：无法用该发布证书上传新包，直到新建 Distribution 证书并更新 Profile。
- **处理**：在低版本或持有私钥的机器新建 `Apple Distribution` → 导出 .p12 双端同步 → 更新 Distribution Profiles → 重新 Archive 上传。

### Q2：Enterprise 内测包闪退

企业分发证一旦被 Revoke，**已安装的 Enterprise 包会立刻失效**。这与开发证无关，必须重新签名分发。

### Q3：Xcode 提示「Choose a certificate to revoke」

说明开发证书已达上限（通常 2 张）。**先查后台哪张在用**，只 Revoke 确定废弃的那张，不要盲选。

### Q4：只有 .cer 没有私钥

`.cer` 无法单独用于签名。必须从有私钥的机器导出 `.p12`，或 Revoke 后重建（仅 Development）。

### Q5：高版本 Archive 报发布证书错误

检查：

1. Distribution 证书是否在钥匙串且含私钥
2. 是否误删了 Distribution
3. Provisioning Profile 是否绑定了正确的 Distribution 证书
