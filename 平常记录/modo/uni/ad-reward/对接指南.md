# Modo-AdReward 用户对接指南

> 面向 **老 uni-app（Vue2/Vue3 + App）** 业务方的接入说明。  
> 插件 ID：`Modo-AdReward` · 穿山甲 GroMore 聚合 SDK · API 与 uni-app x `ad-reward` 对齐。

文档目录：[README.md](./README.md) · API：[API.md](./API.md)

---

## 一、插件能做什么

| 能力 | 页面类型 | 说明 |
|------|----------|------|
| SDK 初始化 | `.vue` / `.nvue` | `initAdSdk` → `startAdSdk` |
| 激励视频 | `.vue` / `.nvue` | load → show → destroy |
| **Banner 嵌入页面** | **`.nvue`（推荐）** | 原生组件 `<modo-ad-banner>` |
| **信息流 Feed 嵌入页面** | **`.nvue`（推荐）** | 原生组件 `<modo-ad-feed>` |
| Banner / Feed（对照） | `.vue` | module API + `bindAdSlot` overlay，**不推荐上线** |
| 开屏 / 全屏 / 插屏 / Draw | `.vue` / `.nvue` | module API（**iOS 当前部分未实现**） |

**平台现状**

| 平台 | 状态 |
|------|------|
| iOS | 激励 ✅ · Banner/Feed 原生组件 ✅ · 其余类型开发中 |
| Android | 激励 ✅ · Banner/Feed 原生组件 ✅（nvue）· 开屏/全屏/插屏/Draw module API ✅ · 离线打包见 [android-实现计划.md](./android-实现计划.md) |

Banner/Feed 为何必须用 nvue，见 [vue与nvue-嵌入式广告方案.md](./vue与nvue-嵌入式广告方案.md)。

## 二、接入前准备

### 2.1 穿山甲媒体平台

1. 注册 [穿山甲媒体平台](https://www.csjplatform.com/)
2. 创建 **iOS / Android 应用**，填写与 App **一致的包名**
3. 创建各类型 **GroMore 聚合广告位**，记下 **广告位 ID（codeId）**
4. 下载对应 SDK（GroMore 融合包）

| 字段 | 含义 | 常见错误 |
|------|------|----------|
| `appId` | 应用 ID | — |
| `appName` | 应用名称 | 与平台登记不一致 |
| `codeId` | **聚合广告位 ID**（多以为 `1` 开头） | 误填 ADN 代码位 ID |

### 2.2 工程要求

- 运行环境：**App（APP-PLUS）**；H5 / 小程序不支持本原生插件
- 将 `nativeplugins/Modo-AdReward` 放入你的 uni 工程
- `manifest.json` → **App 原生插件配置** → 勾选 **Modo-AdReward**
- 拷贝或引用 JS 封装：`js_sdk/ad-reward/index.js`（Demo 用法）

### 2.3 原生代码变更后必须重编 App

修改 `nativeplugins/Modo-AdReward` 内 **原生代码** 后，必须让 App 重新编入最新插件，否则 `uni.requireNativePlugin('Modo-AdReward')` 为空或行为仍是旧版。

- **HBuilderX 云打包 / 自定义基座**：重新制作自定义调试基座 → 安装 → 勾选「使用自定义基座运行」
- **Android 离线打包**（`HBuilder-Integrate-AS`）：同步 `modoAdReward` 模块源码后，在 Android Studio **Rebuild `simpleDemo`** 并安装新 APK（见 [android-实现计划.md](./android-实现计划.md)）
- iOS 离线基座见 [../离线iOS打包-完整协作说明.md](../离线iOS打包-完整协作说明.md)

仅修改 `.vue` / `.nvue` / `js_sdk` **不需要**重编原生（重新发行 www 资源即可）。

## 三、5 分钟快速开始（激励视频）

任意 **`.vue` 或 `.nvue` 页面**均可。

### 3.1 引入

```javascript
import {
  initAdSdk,
  startAdSdk,
  loadRewardAd,
  showRewardAd,
  destroyRewardAd,
  onAdEvent,
  offAdEvent,
  isPluginAvailable
} from '@/js_sdk/ad-reward/index.js'
```

### 3.2 初始化（App 启动后尽早调用）

```javascript
// App.vue onLaunch 或广告页 onLoad
onAdEvent((evt) => {
  console.log('[ad]', evt.adType, evt.phase, evt.ok, evt.errMsg)
})

if (isPluginAvailable()) {
  initAdSdk({
    appId: '你的appId',
    appName: '你的应用名',
    debug: false,        // 上线务必 false
    useMediation: true   // GroMore 聚合
  })
  startAdSdk()
}
```

**顺序固定**：先 `initAdSdk`，再 `startAdSdk`，再 load 任意广告。

### 3.3 播放激励视频

```javascript
// 预加载
loadRewardAd({ codeId: '你的激励广告位ID' })

// 用户点击「看广告」时
showRewardAd()

// 页面卸载或不再需要时
destroyRewardAd()
```

监听 `adType === 'reward'` 的事件：

| phase | 含义 |
|-------|------|
| `load` | 加载完成 |
| `cached` | 视频缓存完成 |
| `show` | 开始展示 |
| `reward` | 发放奖励（业务发奖逻辑写在这里） |
| `close` | 关闭 |
| `error` | 失败 |

页面 `onUnload` 时调用 `offAdEvent(callback)` 解除监听。

## 四、Banner / Feed 嵌入页面（推荐方案）

### 4.1 使用条件

- 页面必须是 **`.nvue`**
- 已完成 **initAdSdk + startAdSdk**
- 已打含原生组件的 **自定义基座**

### 4.2 模板示例

```html
<template>
  <scroller class="page">
    <!-- 列表顶部 Banner -->
    <modo-ad-banner
      v-if="showBanner"
      class="banner"
      :style="{ width: '750rpx', height: bannerHeight + 'px' }"
      :codeId="bannerCodeId"
      :autoLoad="true"
      @sizechange="onBannerSizeChange"
    />

    <!-- 列表内容 … -->

    <!-- 信息流广告位 -->
    <modo-ad-feed
      v-if="showFeed"
      class="feed"
      :style="{ width: '750rpx', height: feedHeight + 'px' }"
      :codeId="feedCodeId"
      :autoLoad="true"
      @sizechange="onFeedSizeChange"
    />
  </scroller>
</template>

<script>
export default {
  data() {
    return {
      bannerCodeId: '你的Banner广告位ID',
      feedCodeId: '你的Feed广告位ID',
      bannerHeight: 150,
      feedHeight: 280,
      showBanner: true,
      showFeed: true
    }
  },
  methods: {
    onBannerSizeChange(e) {
      const detail = e.detail || e
      if (detail.height > 0) this.bannerHeight = detail.height
    },
    onFeedSizeChange(e) {
      const detail = e.detail || e
      if (detail.height > 0) this.feedHeight = detail.height
    }
  }
}
</script>
```

### 4.3 原生组件属性

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `codeId` | String | 是 | GroMore 聚合广告位 ID |
| `autoLoad` | Boolean | 否 | 默认 `true`（对齐 iOS）；`true` 且布局尺寸就绪后自动 load + render |
| `autoShow` | Boolean | 否 | 兼容字段，与 `autoLoad=true` 等效（iOS 无此属性） |
| `debugText` | String | 否 | 调试文案（可选） |

### 4.4 原生组件事件

| 事件名 | 回调参数 | 说明 |
|--------|----------|------|
| `sizechange` / `sizeChange` | `{ height, width }` | **Banner**：占位高度（如 150px）→ render 后回传 SDK 实际高度，在 `@sizechange` 里更新 `bannerHeight`；**Feed（Android）**：固定槽位高度 |

组件内部还会在 `ModoAdRewardEvent` 里上报 `adType: bannerComponent / feedComponent` 的 `load` / `render` / `error` 等 phase，可与 module 广告共用同一 `onAdEvent` 监听。

### 4.5 手动控制加载时机

若希望用户点击后再加载（与 Demo 一致），用 `v-if` 控制挂载，并设 `autoLoad`：

```javascript
data() {
  return { bannerMounted: false, bannerAutoLoad: false, bannerHeight: 150 }
},
methods: {
  onUserClickLoadBanner() {
    this.bannerHeight = 150
    this.bannerAutoLoad = true
    this.bannerMounted = true
  },
  onUserClickShowBanner() {
    this.bannerAutoLoad = true
    this.bannerMounted = true
  },
  onDestroyBanner() {
    this.bannerMounted = false
    this.bannerAutoLoad = false
    this.bannerHeight = 150
  }
}
```

完整示例见工程：`pages/ad-reward/demo-native.nvue`。

### 4.6 不需要传坐标

**不要**向组件传 `left` / `top`。位置与宽高由 **uni 布局**（flex、margin、放在 scroller/list 第几行）决定，与普通 `<view>` 一样写布局即可。

## 五、`.vue` 页面中的 Banner / Feed（不推荐）

仅当你必须留在 `.vue` 且接受 overlay 局限时参考；**产品环境请用 nvue 组件**。

流程概要：

1. 页面放占位 `<view id="banner-slot" :style="{ height: '150px' }" />`
2. `bindAdSlotBySelector(this, '#banner-slot', 'banner')` 绑定槽位
3. `loadBannerAd({ codeId })` → `showBannerAd()`
4. `onPageScroll` 里 `bindAdSlotBySelector(..., { layoutOnly: true })` 同步位置

缺点：快速滑动抖动、易出 `9040008`。详见 [vue与nvue-嵌入式广告方案.md](./vue与nvue-嵌入式广告方案.md)。

Demo 对照：`pages/ad-reward/demo.vue?platform=ios`。

## 六、Module API 一览

以下方法通过 `js_sdk/ad-reward/index.js` 调用；均在 **APP-PLUS** 下生效。

### 6.1 SDK

| 方法 | 参数 | 说明 |
|------|------|------|
| `initAdSdk(options)` | `{ appId, appName?, debug?, useMediation? }` | 保存配置 |
| `startAdSdk()` | — | 启动穿山甲 SDK |
| `isAdSdkReady()` | — | Promise，查询是否已 start |
| `isPluginAvailable()` | — | 是否已加载原生插件 |

### 6.2 激励视频

| 方法 | 参数 |
|------|------|
| `loadRewardAd({ codeId, rewardName?, rewardAmount?, orientation? })` | 聚合广告位 ID |
| `showRewardAd()` | — |
| `destroyRewardAd()` | — |

### 6.3 Banner / Feed（module + overlay，`.vue` 对照用）

| 方法 | 说明 |
|------|------|
| `bindAdSlot('banner' \| 'feed', { left, top, width, height, layoutOnly? })` | 绑定/更新浮层槽位 |
| `bindAdSlotBySelector(vm, '#selector', slot, { layoutOnly? })` | 测量 DOM 后 bind |
| `loadBannerAd({ codeId })` / `showBannerAd()` / `destroyBannerAd()` | Banner 生命周期 |
| `loadFeedAd({ codeId })` / `showFeedAd()` / `destroyFeedAd()` | Feed 生命周期 |

### 6.4 其他类型（API 已导出，iOS 部分待实现）

`load/show/destroy` + `Splash` / `FullScreen` / `Interstitial` / `Draw` — 接口与 uni-app x 一致，iOS 未实现时会收到 `9040005`。

## 七、事件监听

### 7.1 订阅

```javascript
import { onAdEvent, offAdEvent, formatAdEventLog } from '@/js_sdk/ad-reward/index.js'

const onAd = (evt) => {
  console.log(formatAdEventLog(evt))
  // evt: { adType, phase, ok, errCode, errMsg, data }
}

onLoad() {
  onAdEvent(onAd)
},
onUnload() {
  offAdEvent(onAd)
}
```

**注意**：必须使用插件封装的 `onAdEvent`（内部为 `plus.globalEvent`），**不能**用 `uni.$on`。

### 7.2 事件名字段

| 字段 | 说明 |
|------|------|
| `adType` | `sdk` / `reward` / `banner` / `feed` / `bannerComponent` / `feedComponent` / … |
| `phase` | `init` / `start` / `load` / `show` / `render` / `reward` / `close` / `error` / `bind` / … |
| `ok` | 是否成功 |
| `errCode` | 错误码字符串 |
| `errMsg` | 错误描述 |
| `data` | 扩展对象（可选） |

## 八、典型业务场景

### 8.1 App 冷启动

```text
App.onLaunch
  → initAdSdk + startAdSdk
  → （可选）预加载激励 loadRewardAd
```

### 8.2 任务页：点击按钮看激励

```text
用户点击
  → 若未 load：loadRewardAd → 监听 load 成功 → showRewardAd
  → 监听 phase=reward → 发游戏币
  → 监听 close → 刷新 UI
```

### 8.3 资讯列表：顶部 Banner + 中间 Feed

```text
使用 nvue 列表页
  → scroller 内先放 modo-ad-banner，再放列表项，中间插入 modo-ad-feed
  → sizechange 更新高度
  → 快速滑动由原生布局保证不抖
```

### 8.4 主业务是 .vue，只有广告位需要嵌入

```text
方案 A（推荐）：独立 nvue 广告页 / nvue 子模块承载 Banner+Feed
方案 B：Banner 仅固定屏幕底部（overlay 略好，仍非最佳）
方案 C：不用嵌入式 Banner，改用插屏/激励替代
```

## 九、常见错误码

| errCode | 原因 | 处理 |
|---------|------|------|
| `9040001` | SDK 未 start 就 load | 先 `startAdSdk`，或等 start 事件 `ok` |
| `9040002` | appId 为空或未 init | 检查 `initAdSdk` 参数 |
| `9040003` | codeId / slot 为空 | 检查广告位 ID 或 bind 参数 |
| `9040004` | 未 load 就 show | 先 load，监听 load 成功再 show |
| `9040005` | 该广告类型 iOS 未实现 | 换已实现类型或等插件更新 |
| `9040006` | 拿不到 rootViewController | 确保在 App 前台页面调用 |
| `9040008` | overlay 槽位未 bind 或宽高为 0 | 用 nvue 组件可规避；`.vue` 需先 bindAdSlot |
| `9040010` | GroMore 配置文件异常 | 检查插件 Bundles 内 mediation config |

插件未加载时：`isPluginAvailable()` 为 `false` → 需重新制作自定义基座。

## 十二、FAQ

**Q：为什么 `requireNativePlugin` 是 null？**  
A：未勾选 manifest 原生插件，或未用 **自定义基座** 运行。

**Q：Banner 组件创建了但不出广告？**  
A：确认已 `startAdSdk`；组件会在 SDK ready 后自动 retry。看事件里是否有 `bannerComponent` / `wait` / `error`。

**Q：能在 `.vue` 里写 `<modo-ad-banner>` 吗？**  
A：不能稳定工作。请用 `.nvue` 页面。

**Q：和 uni-app x 的 ad-reward 是一套 API 吗？**  
A：JS 方法名、事件 JSON 对齐 `uni_modules/ad-reward/utssdk/interface.uts`；x 用 UTS，老 uni 用 nativeplugins + 本 `js_sdk`。

**Q：Android 什么时候能用？**  
A：见 [android-实现计划.md](./android-实现计划.md)；接入步骤与本文相同，待原生层封装完成即可复用同一套 `js_sdk` 与 nvue 组件名。
