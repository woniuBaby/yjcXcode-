3.`ContentState` 是 Codable，且所有属性都是非可选（`String`、`TimeInterval`）。系统用推送里的 `content-state` JSON 去解码时，只要少一个键，`Decodable` 就会解码失败，整次更新就不会生效。

所以：缺一个字段 = 无法远程更新灵动岛，这是当前代码导致的。

要让推送不必带全字段（例如可以省略颜色或部分文案），需要改解码逻辑：在 ActivityAttributesModel 里为 `ContentState` 实现自定义 `init(from decoder:)`，对缺失的键用默认值，这样服务端就可以只传要更新的字段。

```swift
  /// 远程推送解码：缺失的 key 用默认值，服务端不必传全字段
  public init(from decoder: Decoder) throws {
      let c = try decoder.container(keyedBy: CodingKeys.self)
      ID = try c.decodeIfPresent(String.self, forKey: .ID) ?? ""
      nameStart = try c.decodeIfPresent(String.self, forKey: .nameStart) ?? ""
      nameEnd = try c.decodeIfPresent(String.self, forKey: .nameEnd) ?? ""
    btnTitleStart = try c.decodeIfPresent(String.self, forKey: .btnTitleStart) ?? ""
		...
}
```
