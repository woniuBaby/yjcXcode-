# 01 产品与能力说明

## 1. 产品目标

在 iOS 桌面提供与游戏状态一致的小组件展示，覆盖：

| 业务 | 尺寸 | 核心展示 |
|------|------|----------|
| 种花 | Small / Medium | 种植进度与倒计时、花艺出售、水滴圆环与恢复倒计时 |
| 珍珠采集 | Small / Medium | 珍珠数、雇员槽位倒计时、按钮文案 |
| 公会竞赛 | Small / Medium | 积分、任务、进度条、活动结束时间 |

用户添加小组件后，主 App 在关键状态变化时更新 App Group 快照并触发 WidgetKit 刷新；Extension 只读快照渲染，不直连游戏服。

## 2. 能力边界

### 2.1 种花（v3）

- **Medium**：三等分竖栏 — 左种植 `flower_plant`、中花艺 `flower_sale`、右水滴 `flower_water_drop`（竞赛栏 `flower_competition` 在协议中保留，当前 Medium UI 以三栏为主）。
- **Small**：仅种植栏，紧凑倒计时（`compactCountdownOnly`）。
- **状态驱动**：每栏 `status`（0 空闲 / 1 进行中 / 2 完成）+ `topTxt` / `bottomTxt` / `img` / `timeBar` / `bar`（水滴）。
- **无业务数据**：显示 Extension 内置占位（中英文由系统首选语言决定，非地区码）。

### 2.2 珍珠 / 公会

- 独立快照文件与独立 `*Mgr`，JS 走 `widget/pearl`、`widget/guild`（或项目内等价 opt）。
- 与种花 v3 **不共用** `widget_snapshot.json` 字段。

### 2.3 不做 / 已移除

- 种花 **Large** 尺寸（已下线）。
- 从灵动岛 Activity 自动同步到桌面快照（已解耦）。
- 旧版扁平快照字段（`plantPhase`、`completedIconUrl` 等顶层字段）— 写入会被拒绝。

## 3. 与灵动岛的关系

- 灵动岛：ActivityKit，长驻活动条，推送/活动内 update。
- 小组件：WidgetKit，冷启动与 Timeline 驱动展示。
- **数据不自动打通**：若产品需要一致，须游戏侧分别调用对应能力 API。

## 4. 用户可见行为

| 场景 | 预期 |
|------|------|
| 首次添加、从未写入快照 | 占位文案 + 默认插图（中/英） |
| 游戏推送 v3 单栏 | 对应栏更新；其它栏保留上次 merge 结果 |
| 种植/出售倒计时结束 | Timeline 到点切换 UI，无需再次 request |
| 水滴恢复 | 圆环按 `timeBar` 滚动，到点 thumb+1 |
| 主 App 未配置 App Group | 快照失败，小组件保持空态 |
| 传非 v3 包 | `status: fail`，message 提示必须 v3 |

## 5. 验收标准（摘要）

- [ ] 主 App 与 Extension App Group、entitlements 一致。
- [ ] `widget/default` request/update 使用 `{ type, data }`，磁盘 `widget/widget_snapshot.json` 为 schema v3。
- [ ] Medium 三栏、Small 单栏与游戏状态一致；倒计时到点自动变完成态。
- [ ] 无快照时中英文占位符合系统语言；有快照时以服务端文案为准。
- [ ] 珍珠、公会各自快照刷新对应 kind。
- [ ] 真机添加/删除小组件、杀进程冷启动后展示仍正确。
